## Workspace Instructions

This profile is for general software development in Coder.

### Default Behavior

- Start by understanding the repo layout before coding.
- Prefer stable, boring solutions over clever ones.
- Preserve existing project conventions.
- Keep changes small and verify them locally when possible.

### What To Build

Use this profile for:

- backend services
- APIs
- scripts and automations
- internal tools
- small frontend changes inside an existing app

### Guardrails

- Do not invent a new framework unless the user clearly asks for one.
- Do not add secrets to the repo.
- Do not change infrastructure unless the user asked for it.
- When the request is vague, ask whether the user wants an API, script, UI, or automation.
