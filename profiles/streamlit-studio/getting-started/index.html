Welcome

You can build a Streamlit app here even if you have never used Coder or Codex before.

This workspace is set up so you can upload files, describe what you want in plain English, and let Codex build the first version for you. You do not need to know where everything lives yet. Start with the steps below and use the page as your map.

Start here

  • Use File Browser to bring files into the workspace.
  • Open Terminal and run codex.
  • Describe the app you want in normal language.
  • Open My Apps to see the result users will actually use.
  • Open VS Code if you want to inspect or edit files directly.

Two commands to remember

Start a new Codex session:
codex
Return to the most recent Codex session:
codex resume --last

Follow These 4 Steps

If you are new to the workspace, do these in order the first time.

1

Get your files into the workspace

Click File Browser. Upload raw files into ~/workspace/uploads. Keep notes in ~/workspace/notes, put reference material in ~/workspace/reference, and save finished exports in ~/workspace/outputs.

If you are not sure where a file should go, put it in uploads/ first. That is the safest default.
2

Start Codex from the terminal

Click Terminal, then type codex. That opens an interactive Codex session inside the workspace. If you were already working and want to continue, use codex resume --last.

You do not need to write code first. Just tell Codex what the app should do, who it is for, what files to use, and what the most important screens or actions are.
3

Describe the app in plain language

Be specific about the audience, the data source, and the actions users need to take. The more concrete the prompt, the better the first version will be.

Build a Streamlit dashboard for the admissions team. Use the CSV files in ~/workspace/uploads. The dashboard should let users filter by campus and term, show top-level totals, and export filtered results. Make the design clean and easy for non-technical staff to use.
4

Review the result in My Apps

Click My Apps to open the user-facing Streamlit app. If something is unclear or missing, go back to the terminal and tell Codex exactly what to change. Use VS Code when you want to inspect files directly.

What each workspace button is for

These are the buttons most people will use every day in this profile:

  • Getting Started: this page and the orientation materials.
  • My Apps: the Streamlit app studio and user-facing output.
  • File Browser: the clean file area for uploads, notes, references, and outputs.
  • VS Code: the editor for files, folders, and project code.
  • Terminal: where you run Codex and shell commands.

Where the code goes

In this Streamlit profile, app code usually lives in ~/my-apps. Uploaded files and user-managed documents live in ~/workspace. That separation helps keep app code away from raw uploads and exported deliverables.

~/workspace/uploads

Raw CSV, Excel, PDF, image, and other input files.

~/workspace/outputs

Exports, generated files, finished reports, and deliverables.

~/workspace/notes

Working notes, drafts, summaries, and decisions.

~/my-apps

The Streamlit app studio and the code Codex builds.
